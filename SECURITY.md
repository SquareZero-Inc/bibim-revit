# Security Policy

## Reporting a vulnerability

**Do not open a public issue for security vulnerabilities.**

Email: security@squarezero.kr

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

We'll acknowledge within 3 business days and aim to ship a fix within 14 days depending on severity.

## Scope

This policy covers the `SquareZero-Inc/bibim-revit` codebase.

The BIBIM add-in handles:
- Your Anthropic API key (stored locally in `%AppData%\BIBIM\rag_config.json`)
- Active Revit document data passed as context to the Claude API
- C# code generated by Claude and executed inside Revit via `ExternalEvent`

None of this data leaves your machine except for API calls to `api.anthropic.com`.

## Out of scope

- Issues in third-party dependencies (report upstream)
- API key leakage caused by user sharing their own `rag_config.json`
